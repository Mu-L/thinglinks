# 前端开发规范

## 目录结构规范

### 必须遵守的三级目录原则

此规范适用 不限于 components、api、routes、assets、images、views、locales 等等
为了快速定位功能所属应用及服务模块，请严格遵守此规范。

#### 应用名称

```properties
格式要求：全小写字母+数字（禁止特殊字符）
示例：
- iot（物联网业务系统平台统一应用名）
- card（物联网卡业务系统平台统一应用名）
```
#### 服务名称

```properties
格式要求：与后端微服务名称严格一致
典型值：
- system   ▸ 系统服务
- link     ▸ 物联服务
- rule     ▸ 规则引擎
```

#### 模块名称

```properties
格式要求：使用业务术语（禁用技术术语）
┌───────────────┬─────────────────────────────┐
| 业务场景       | 推荐命名                     |
├───────────────┼─────────────────────────────┤
| 设备OTA升级   | ota                         | 
| 产品管理       | product                     |
| 实时监控       | monitor                     |
└───────────────┴─────────────────────────────┘

```


### 强制规则

- 服务名称必须与后端 spring.application.name 中配置的服务名完全一致
- 模块目录内禁止出现common/util等通用性目录
