# 服务业务场景说明

## 消息中间件场景说明

本工程中使用了 Kafka、RocketMQ 和 RabbitMQ 三种主流的消息队列服务,它们各有特点,适用于不同的场景需求。
例如：其他平台或厂商通过数据规则引擎 推送至消息中间件的业务数据，可以更加快速的开发处理。

### Kafka

设计原理:

- 基于发布-订阅模式的分布式流处理平台,重点关注高吞吐量的数据流处理。

典型应用:

- MQTT 设备的海量数据实时处理。
- TCP 消息的实时处理。
- WebSocket 消息的实时处理。
-

### RocketMQ

设计原理:

- 基于发布-订阅模式的分布式消息中间件,专注于可靠性和可扩展性。

### RabbitMQ

设计原理:

- 基于AMQP协议的消息代理,适用于企业级应用中的常见消息场景。
- 业务解耦、异步处理。

典型应用:

- 基于 RabbitMQ 广播实现分布式 WebSocket（集群上下线、集群消息等）
- 用于系统的异步业务逻辑处理
